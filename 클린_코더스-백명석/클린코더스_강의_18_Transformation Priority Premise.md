1. Transformation Priority Premise
  * 테스트 성공기법의 우선순위
  * 실패한 테스트를 성공시킬때도 쉬운것 부터 해라
  * Transformation
    - 구조 변경없이 행위를 변경하는 것
    - 구조 변경없이 행위 변경을 할 수 없다.
    - '구조변경이 없다'를 '주목 할 만한 구조변경' 없이로 이해하라.
    - 실패하는 테스트가 있을때
  * Refactoring
    - 행위 변경없이 구조를 변경하는 것
